# 기프티콘 consume 중복 요청(멱등/조건부 업데이트)


### 상황

- 사용 처리 API가 네트워크 재시도로 두 번 호출됨
- 또는 동일 기프티콘을 2명이 동시에 사용 시도(운영 실수/공유)

### 토론 포인트

- “consume은 멱등하게 할까, 409로 막을까?”
- 상태 조건부 업데이트(WHERE status=ISSUED)
- 감사로그/이벤트(사용 기록 추적)


### 구현 검증(테스트)

- 동시 consume 2요청 → 1 성공, 1 실패(혹은 멱등 정책이면 둘 다 200이지만 상태 변화는 1번)
- API 설계 감각(멱등 vs 에러) 토론에 좋음.
